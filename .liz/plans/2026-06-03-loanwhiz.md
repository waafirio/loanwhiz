# LoanWhiz

## Context & intent

### The problem

Structured finance teams repeatedly rebuild the same AI foundations from scratch. A bank wants an investor Q&A agent. A servicer wants cashflow anomaly explainers. A rating agency wants a prospectus extractor. Each team builds the same plumbing — data connectors, parsers, validators, grounding patterns, audit logs — one project at a time. Reuse is limited, governance is inconsistent, and promising AI prototypes struggle to reach production.

This is the problem the Barcelona AI Tinkerers Structured Finance Hackathon 2026 (Challenge 1) is asking teams to solve. Demo day is 10 June 2026. Seven days from plan capture.

### Why this shape

The plan is organised around a core insight that emerged during design: the most defensible and differentiating thing we can build is a **computational deal model** — a machine-executable representation of a structured finance deal derived from its own documentation. This goes beyond document Q&A or tape analytics in isolation. It connects the prospectus (deal rules), the loan tape (actual performance), and the investor report (reported outcomes) into a causally coherent model that can:

- Run the waterfall from first principles and compare to what the servicer reported
- Monitor covenant and trigger compliance over time
- Project forward cashflows under stress scenarios
- Serve as a shared knowledge artifact for all downstream agents and clients

This "deal model" approach was chosen over alternatives (generic RAG, simple document Q&A, standalone tape analytics) for three reasons:

1. **Judges will recognise it.** Samuel Griek (Eagleyes.ai, judge) builds TrancheFlow — a commercial cashflow model builder. He will immediately distinguish a real waterfall model from a shallow demo. Mark Mearing-Smith (Jefferies CLO desk, judge) spent years at Trepp building CLO analytics. Luca Borella (Algoritmica.ai, organiser, judge) co-founded deeploans — he wants to see it extended, not ignored.

2. **It collapses three of the eight "extra inspiration" use cases** from the challenge spec into one coherent demo: Prospectus Extractor + Cashflow Anomaly Explainer + Covenant Compliance Agent.

3. **It is genuinely novel.** No other team is likely to extract a machine-executable waterfall from a prospectus PDF and run it against real tape data in 7 days. The extraction pipeline (Docling → section routing → definitions graph → Gemini structured extraction → executable waterfall object) was validated against the Green Lion 2026-1 prospectus: section 5.2 (Revenue Priority of Payments) extracted cleanly as 11 ordered, conditionally-specified payment steps.

### What we are building on

- **deeploans** (Algoritmica.ai, Apache 2.0) — ESMA loan-level ETL framework with FastAPI backend and MCP server. Our ESMA tape normaliser extends this rather than replacing it. Directly relevant to Luca Borella.
- **FINOS AI Governance Framework** — templates, risk patterns, mitigation patterns. Used for the governance layer: model cards, confidence scoring, audit trails, human review routing.
- **Green Lion 2026-1** (Algoritmica/green-lion-2026, HuggingFace) — the only publicly available complete deal package (prospectus + 3 monthly ESMA tapes + 3 monthly investor reports). This is the primary test dataset.
- **Docling** (IBM, Apache 2.0) — structure-aware PDF extraction. Validated: 77 tables extracted, waterfall section located at char 699,437, full Revenue Priority of Payments (11 steps) extracted as clean structured lists.
- **LangGraph** — agent orchestration framework explicitly recommended in the challenge spec.
- **Vertex AI / Gemini 2.0 Flash** — 1M token context, fast, GCP credits available.

### Architecture

```
CLIENTS: Chat/Q&A | Dashboard | Compliance | REST API
         ↓
LANGGRAPH AGENT SERVICE
  Planner → DAG executor → Validator → Confidence scorer
  Human review router → Audit trail
  [Governance: FINOS AI Governance Framework patterns]
         ↓                        ↓
SF PRIMITIVES              DEAL MODEL (JSON, per deal)
  waterfall_runner            definitions{}
  covenant_monitor            waterfall[] ← section 5.2
  report_verifier             triggers[]
  cashflow_projector          tranches[]
  audit_logger                pool_eligibility{}
         ↓                        ↑
         └──── DATA LAYER ────────┘
  deeploans ETL + MCP server (ESMA tape ingestion, multi-annex)
  Docling extraction pipeline (prospectus → deal model JSON)
  HuggingFace: Algoritmica/green-lion-2026
```

### Tech stack

- Python (LangGraph, Vertex AI SDK, Docling, pandas, pytest)
- TypeScript MCP server (existing sf-mcp code, now superseded as primary artefact but retained as open-source companion)
- Vertex AI Gemini 2.0 Flash for LLM extraction and agent reasoning
- GCP credits for compute
- Apache 2.0 license, FINOS incubation pathway

### Alternatives considered and rejected

- **MCP server only (TypeScript)**: Built first, but challenge spec targets LangGraph/AutoGen/CrewAI. MCP is not mentioned. Judges expect Python orchestration. MCP server retained as open-source companion but not the primary submission.
- **Anthropic API**: User has Claude Max subscription but not a separate Anthropic API key. GCP credits available → Vertex AI is the correct path.
- **Standard RAG**: Fails on prospectuses due to cross-references (defined terms, cross-section dependencies). Replaced by section routing + definitions graph + targeted long-context extraction.
- **GraphRAG**: Designed for multi-document corpora, not single-document deep extraction. Overkill for single-deal analysis. Would add value for cross-deal comparison post-hackathon.
- **Challenge 2 (World Model for Credit)**: Requires Python/ML expertise, PyTorch, RAPIDS, CUDA. Only 3 monthly tape snapshots available — insufficient for sequence model training. Ruled out.
- **Challenge 3 (Asset-Backed Technology)**: No provided data, poorly scoped. Ruled out.

### Demo target (10 June)

Load the Green Lion 2026-1 deal. Show:
1. Prospectus extracted → deal model JSON (waterfall, triggers, tranches, definitions)
2. Three monthly ESMA tapes loaded → pool statistics, arrears trend, EPC breakdown
3. Waterfall executed against tape collections → computed distributions per tranche
4. Computed distributions vs investor reports → verification (match/mismatch flagged)
5. Covenant monitor → trigger proximity over time
6. Forward projection → 12-month cashflow scenario
7. Chat interface → "What happens to Class B if defaults double?" answered with citations

## Decomposition

### Epic: Foundation & Data Layer   (umbrella #TBD)

Establish the Python project structure, deeploans integration, Vertex AI connection, and ESMA tape normaliser primitive. This is the base everything else builds on.

- **Python project scaffold** — Set up Python package structure, LangGraph dependency, Vertex AI SDK, pytest, linting. Sequencing: parallel.
- **deeploans integration** — Wire deeploans FastAPI backend and MCP server as the canonical data source for ESMA loan tapes. Document the integration contract. Sequencing: parallel.
- **ESMA tape normaliser primitive** — Extend deeploans with pool analytics: weighted averages (balance-weighted), arrears breakdown (multi-bucket), EPC/geographic/rate-type distributions. Multi-annex schema detection (Annex 2–8). Typed input/output contract, confidence score, citations. Sequencing: sequential. After deeploans integration.
- **Primitive interface spec** — Define the base Primitive class: typed inputs, typed outputs, version, confidence score, citation format, audit log entry. All subsequent primitives implement this interface. Sequencing: parallel.

### Epic: Deal Model Extraction Pipeline   (umbrella #TBD)

Build the pipeline that turns a prospectus PDF into a structured, machine-executable deal model JSON artifact. This is the technical centrepiece.

- **Docling section router** — Parse prospectus markdown output into a section tree using TOC + regex header detection. Map section titles to character ranges. Locate key sections: Definitions, Priority of Payments, Credit Enhancement, Conditions of the Notes, Eligibility Criteria. Sequencing: parallel.
- **Definitions graph extractor** — Extract all defined terms from the Definitions section into a key-value store (term → definition + page citation). Resolve cross-references when extracting other sections. LLM call: Gemini, targeted section only. Sequencing: sequential. After Docling section router.
- **Waterfall extractor** — Extract Revenue Priority of Payments, Redemption Priority of Payments, and Post-Enforcement Priority of Payments as ordered, conditionally-specified Python-executable objects. Resolve "Available Distribution Amount" and similar references via definitions graph. LLM call: Gemini, section 5.2 only (~8 pages). Output: structured waterfall JSON with citations per step. Sequencing: sequential. After definitions graph extractor.
- **Covenant & trigger extractor** — Extract trigger definitions, threshold values, monitoring conditions (e.g. cumulative loss rate > X%, PDL breach). Output: structured trigger objects with threshold, metric, direction, citation. Sequencing: sequential. After definitions graph extractor.
- **Deal model assembler** — Combine all extracted components into a single deal_model.json artifact per deal. Validate completeness. Cache to disk — extraction runs once per deal. Sequencing: sequential. After waterfall extractor and covenant extractor.

### Epic: SF Primitives   (umbrella #TBD)

Build the computational primitives that operate on the deal model and loan tape data. These are the reusable, versioned components the framework offers.

- **Waterfall runner** — Execute the extracted waterfall against monthly tape collections. Inputs: deal model waterfall, monthly collections (interest, principal, prepayments, recoveries). Output: computed distributions per tranche per period, with full audit trace. Sequencing: parallel.
- **Covenant monitor** — Check tape metrics against extracted trigger thresholds. Track proximity over time (% of threshold). Flag breaches and near-misses. Output: compliance status per trigger per period, with citations to prospectus definitions. Sequencing: parallel.
- **Report verifier** — Compare waterfall-computed distributions against investor report actuals. Flag discrepancies. Output: verification result per line item, match/mismatch, delta. Sequencing: sequential. After waterfall runner.
- **Cashflow projector** — Project forward cashflows under base and stress scenarios (e.g. 2× default rate, rate shift). Uses waterfall runner internally. Output: 12-month projection per tranche, scenario comparison. Sequencing: sequential. After waterfall runner.
- **Audit logger** — Wrap every primitive call with provenance: input hash, output, confidence score, citations, timestamp, model version, human review flag. Follows FINOS AI Governance Framework patterns. Sequencing: parallel.

### Epic: LangGraph Agent Service   (umbrella #TBD)

Build the orchestration layer that composes primitives into DAGs to answer structured finance questions. Expose as a REST service with multiple client surfaces.

- **Primitive tool wrappers** — Wrap each SF primitive as a LangGraph tool with typed schemas. Sequencing: parallel.
- **Planner agent** — LangGraph agent that takes a natural language question, selects relevant primitives, and builds an execution DAG. Backed by Gemini 2.0 Flash. Sequencing: sequential. After primitive tool wrappers.
- **DAG executor with validation** — Execute the planned DAG, validate outputs at each step, retry on low-confidence results, route to human review when confidence below threshold. Sequencing: sequential. After planner agent.
- **REST API** — FastAPI service wrapping the agent. Endpoints: POST /query, GET /deal/{id}/model, GET /deal/{id}/compliance, POST /deal/{id}/project. Sequencing: sequential. After DAG executor.
- **Governance evidence pack** — Per-query: confidence scores, citation trail, tool call audit log, replayable reasoning trace. Model card for the agent. Follows FINOS AI Governance Framework templates. Sequencing: parallel.

### Epic: Client Applications   (umbrella #TBD)

Build the client interfaces that demonstrate the framework in action on the Green Lion deal.

- **Chat interface** — Simple web UI for natural language Q&A against a deal. Calls REST API. Streams responses. Shows citations inline. Sequencing: parallel.
- **Deal dashboard** — Visualise: waterfall distributions per tranche per period, pool performance trend (arrears, defaults, EPC), covenant status (green/amber/red per trigger). Calls REST API. Sequencing: parallel.
- **Compliance analysis view** — Report verifier output: computed vs reported, discrepancies flagged. Covenant monitor: proximity to breach over time. Sequencing: sequential. After report verifier and covenant monitor.

### Epic: Demo, Documentation & Governance   (umbrella #TBD)

Prepare the demo script, open-source documentation, FINOS governance artefacts, and primitive catalogue. Required deliverables per the challenge spec sprint plan.

- **Green Lion demo script** — End-to-end walkthrough: extract deal model → load tapes → run waterfall → verify reports → monitor covenants → project scenarios → answer chat questions. Reproducible, runnable. Sequencing: parallel.
- **Primitive catalogue** — Document each primitive: purpose, typed interface, example inputs/outputs, confidence scoring logic, known limitations. The "contribution guide" for new primitives. Sequencing: parallel.
- **Model card & governance document** — Per FINOS AI Governance Framework templates: model card for the extraction pipeline, governance pattern document, data card for Green Lion dataset usage. Sequencing: parallel.
- **README & quickstart** — Setup instructions, architecture diagram, how to run against a new deal, how to contribute a new primitive. Sequencing: parallel.
